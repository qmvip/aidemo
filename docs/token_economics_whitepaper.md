## 一、项目概述

### 1.1 望易AI区块链定位

望易AI区块链（Wangyi AI Blockchain）是一个**分布式AI计算网络**，旨在构建下一代去中心化AI基础设施。项目采用**N2N（节点到节点）+ P2P（点对点）双模式网络架构**，结合V5决策引擎，实现高效、安全、低成本的分布式AI计算服务。

### 1.2 核心技术创新

- **双模式网络**：N2N模式支持高速内部通信，P2P模式支持去中心化节点发现与组网
- **V5决策引擎**：多维任务分拆 + 幂律加权融合的 consciousness fusion 算法
- **量子记忆同步**：跨节点的状态与记忆增量同步
- **分布式自醒机制**：节点自主发现、加入、协同工作

### 1.3 市场规模与机会

根据预测，2026年全球AI算力市场规模将突破**500亿美元**，去中心化算力赛道（DePIN）正值爆发期。中心化云服务（AWS、GCP、Azure）存在成本高、隐私泄露、垄断等痛点，为去中心化AI算力网络创造了巨大的市场机会。

## 五、通胀与通缩机制（价值支撑核心）

### 5.1 通胀模型（产出机制）

望易AI区块链采用**算力挖矿产出 + 减半递减 + 动态调节**机制，激励早期参与者和长期持有者。

#### 5.1.1 挖矿产出规则

```
年通胀总量 = 基础通胀 + 算力贡献奖励

基础通胀池：4.5亿WY（45%分配），按减半周期释放
算力贡献奖励池：按全网算力贡献度分配
```

#### 5.1.2 减半周期（核心规则）

| 阶段 | 周期 | 年通胀量 | 累计释放 |
|------|------|----------|----------|
| 第1期 | 第1-2年 | 8000万/年 | 1.6亿 |
| 第2期 | 第3-4年 | 4000万/年 | 2.4亿 |
| 第3期 | 第5-6年 | 2000万/年 | 2.8亿 |
| 第4期 | 第7-8年 | 1000万/年 | 3.0亿 |
| 第5期及以后 | 第9年+ | 500万/年 | 恒定3.05亿 |

**减半触发**：每2年减半，与比特币类似，确保稀缺性

#### 5.1.3 动态调节机制（供需平衡）

根据网络使用率动态调节奖励与销毁比例：

| 全网算力使用率 | 奖励调节 | 销毁调节 |
|----------------|----------|----------|
| **< 50%（供过于求）** | ×0.8 | ×1.2 |
| **50%-80%（正常）** | ×1.0 | ×1.0 |
| **> 80%（供不应求）** | ×1.2 | ×0.8 |

**调节目的**：
- 供过于求时：减少产出、增加销毁，防止代币贬值
- 供不应求时：增加产出、减少销毁，激励更多节点加入

## 5.5 算力经济模型（核心商业模式）

### 5.5.1 算力定价机制

望易AI区块链采用**基础价 + 竞价溢价**双层定价机制：

#### 基础定价公式

```
算力单价（USD/小时） = 基础成本 × 供需系数 × 质量系数

基础成本：GPU型号基准成本（如A100 = $0.5/小时）
供需系数：全网空闲率反向调节（空闲率越高，价格越低）
质量系数：节点信用评级（0.8-1.5）
```

#### 定价模式

| 模式 | 说明 | 适用场景 |
|------|------|----------|
| **固定价** | 基础价±10%浮动 | 稳定需求、长期合同 |
| **竞价拍卖** | 出价高者得 | 紧急算力、峰值需求 |
| **动态调节** | 根据供需自动调价 | 实时算力交易 |

#### 支付方式

- **WY支付**：60%（享受5%折扣）
- **USDT/法币**：40%（原价）
- **企业客户**：支持月结、季度结

### 5.5.2 算力奖励池分配

#### 激励池来源

```
月激励池 = 网络当月收入的30% + 通胀代币的50%

示例：月收入100万USDT → 30万进入激励池
```

#### 分配公式

```
节点月奖励 = 基础分配 × 算力占比 × 在线系数 × 质押权重

基础分配 = 激励池 ÷ 参与节点数
算力占比 = 节点算力 ÷ 全网算力
在线系数 = 实际在线时长 ÷ 理论时长（最高1.2）
质押权重 = sqrt(质押量 ÷ 最低质押)（最高2.0）
```

#### 分配比例

| 分配对象 | 比例 | 说明 |
|----------|------|------|
| 算力贡献者 | **70%** | 按算力贡献分配 |
| 存储贡献者 | **10%** | 数据存储节点 |
| 带宽贡献者 | **10%** | 网络传输节点 |
| 生态建设者 | **10%** | 开发者、推广者 |

### 5.5.3 供需失衡应对

| 情况 | 应对措施 |
|------|----------|
| **算力过剩（使用率<50%）** | 奖励×0.8，销毁×1.2，价格下调10% |
| **供需平衡（使用率50-80%）** | 正常运营 |
| **算力短缺（使用率>80%）** | 奖励×1.2，销毁×0.8，价格上调20% |
| **极端短缺（使用率>95%）** | 紧急扩容奖励+优先调度 |

## 5.6 数据激励模型（AI核心价值）

望易AI区块链的数据是网络核心资产，数据贡献者是生态重要组成部分。

### 5.6.1 数据类型与质量分级

| 数据类型 | 质量等级 | 基础奖励系数 | 说明 |
|----------|----------|--------------|------|
| **原始数据** | L1 | ×1.0 | 未经处理的原始数据 |
| **清洗数据** | L2 | ×2.0 | 去噪、去重、格式标准化 |
| **标注数据** | L3 | ×5.0 | 人工/AI标注，含标签 |
| **脱敏数据** | L4 | ×10.0 | 隐私处理后的高质量数据 |

### 5.6.2 数据奖励计算

```
数据贡献奖励 = 数据量 × 质量系数 × 需求系数 × 使用次数

数据量：GB/TB为单位
质量系数：L1=1, L2=2, L3=5, L4=10
需求系数：热门领域×1.5，冷门×0.5
使用次数：数据被调用次数累积
```

### 5.6.3 数据交易分成

当数据被付费使用时，收益分配：

| 角色 | 比例 | 说明 |
|------|------|------|
| **数据贡献者** | **70%** | 数据所有者 |
| **网络运营** | **20%** | 存储、带宽、平台 |
| **生态基金** | **10%** | 开发者激励 |

### 5.6.4 数据质量验证

- **自动验证**：AI模型检测数据质量、重复率、噪声
- **人工抽检**：社区志愿者抽检，准确率纳入贡献者评分
- **惩罚机制**：数据造假者扣除奖励，情节严重者封号

## 六、质押与收益机制

### 6.1 质押参数（明确门槛）

| 参数 | 主节点 | 计算节点 | 备注 |
|------|--------|----------|------|
| **最低质押量** | 50,000 WY | 5,000 WY | 低于此无法参与 |
| **推荐质押量** | 100,000 WY | 10,000 WY | 获得完整收益 |
| **质押周期** | 30天（最短） | 30天（最短） | 可随时增加质押 |
| **解押冷却期** | 21天 | 21天 | 冷却期内无收益 |
| **年化收益率** | **15-20%** | **8-12%** | 基准收益 |
| **治理分红** | +5% | +3% | 参与治理额外收益 |

### 6.2 收益计算公式

```
节点月收益 = 基础收益 + 算力贡献奖励 + 活跃度奖励 + 治理分红

基础收益 = 质押量 × (年化收益率 ÷ 12)
算力贡献奖励 = (个人算力 ÷ 全网算力) × 当月激励池 × 50%
活跃度奖励 = 在线时间占比 × 基础收益 × 20%
治理分红 = 参与投票次数 ÷ 总提案数 × 基础收益 × 5%

激励池 = 网络当月收入的30% + 通胀代币的50%
```

**收益分配周期**：每月1日结算上月收益，自动发放至钱包

**示例计算（主节点，质押10万WY）**：
```
基础收益 = 100,000 × 18% ÷ 12 = 1,500 WY/月
假设贡献1%算力，激励池1000万WY：
算力奖励 = (1% ÷ 100%) × 1000万 × 50% = 50,000 WY/月
活跃度 = 99% × 1500 × 20% = 297 WY/月
治理分红 = 参与5个提案 × 1500 × 5% = 375 WY/月
----------------------------------------
总计：≈ 53,172 WY/月 ≈ 年化63.8%
```

### 6.3 惩罚机制（清晰可量化）

| 违规行为 | 惩罚措施 | 恢复条件 |
|----------|----------|----------|
| 节点离线（24小时内） | 扣除当次收益的**10%** | 无 |
| 节点离线（24-72小时） | 扣除当次收益的**30%** | 重新上线 |
| 节点离线（72小时以上） | 扣除质押的**5%**，停止收益 | 重新上线+等待7天 |
| 算力造假（虚报算力>10%） | 扣除质押的**20%** | 严重违规 |
| 恶意行为（攻击、双花） | **扣除全部质押**，永不释放，IP封禁 | N/A |
| 数据造假 | 扣除质押的**15%**，驱逐网络 | N/A |

### 7.2 投票权重设计（防垄断核心）

**基础投票权**：
- 每持有1,000 WY = 1基础投票权

**长期质押加权**（避免大户垄断）：
| 锁仓时间 | 投票权重 | 说明 |
|----------|----------|------|
| < 90天 | ×1.0 | 基础权重 |
| 90-180天 | ×1.3 | 长期持有激励 |
| 180-365天 | ×1.6 | 核心社区 |
| > 365天 | ×2.0 | 最高信任 |

**节点贡献加权**：
- 主节点：额外×1.5投票权
- 计算节点：额外×1.2投票权

**最终投票权 = 基础 × 锁仓加权 × 节点加权**

**Quorum定义**：实际投票权 ÷ 总流通投票权

### 7.4 治理流程（14天完整周期）

```
1. 提案提交（1天）
```

## 八、收入模型

### 8.1 六大盈利点

#### 8.1.1 算力交易抽成（核心现金流）

- **模式**：连接算力供给方（节点）与需求方（AI公司、开发者、企业）
- **盈利**：每笔算力交易抽成**5-15%**
- **市场规模**：AI算力需求爆发，网络成本比AWS/GCP低70-90%

#### 8.1.2 节点服务/管理费用（稳定收入）

- **模式**：为企业/机构提供专属节点集群、SLA保障、优先调度、安全隔离
- **盈利**：月费/年费订阅（企业级客户）
- **案例**：Render、io.net企业级订阅服务

#### 8.1.3 API/SDK授权（生态变现）

- **模式**：向第三方开发者/项目收取API调用费、SDK授权费、商业使用许可费
- **盈利**：按调用次数/月费/分成收费
- **优势**：N2N+P2P双模式接口是稀缺入口

#### 8.1.4 代币经济+质押收益（Web3标配）

- **模式**：节点质押、算力贡献、生态贡献获得代币奖励
- **盈利**：代币交易手续费、质押抽成、治理费用

#### 8.1.5 AI模型推理/训练服务（高毛利）

- **模式**：基于分布式网络提供低成本AI模型推理、微调、训练服务
- **盈利**：按算力/按任务收费，**毛利60-80%**

#### 8.1.6 数据/隐私服务（高壁垒）

- **模式**：提供分布式数据存储、隐私计算、联邦学习服务
- **盈利**：数据存储费、隐私计算服务费、数据交易分成

## 九、估值模型

### 9.1 估值逻辑

基于**梅特卡夫定律（Metcalfe's Law）**和**网络效应**：

```
网络价值 = 节点数² × 单节点价值
```

| 阶段 | 节点数 | 估值（万美元） | 逻辑 |
|------|--------|----------------|------|
| 种子轮 | 1-10 | 2000-3000 | MVP阶段，网络价值低 |
| 扩展期 | 50 | 3000-5000 | 初步网络效应 |
| 成长期 | 100 | 5000-1亿 | 强网络效应 |
| 规模化 | 500 | 1亿-3亿 | 生态爆发 |
| 成熟期 | 1000+ | 3亿-10亿 | 行业龙头 |

### 9.2 估值支撑

**100节点→2亿美元估值的支撑**：

1. **计算力聚合**：100节点 = 分布式超算网络，年算力价值超1亿美元
2. **数据网络效应**：更多节点 = 更多训练数据 = 更高AI能力
3. **生态价值**：正向循环，节点即"矿工"
4. **P/S倍数**：成长期DePIN项目P/S = 10-20倍
